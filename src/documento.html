📅 UniAgenda

Documentação Técnica do Sistema

📌 Visão Geral

O UniAgenda é um sistema web desenvolvido em React para gerenciamento de eventos acadêmicos. Ele permite que diferentes perfis de usuários (alunos, responsáveis, professores e coordenadores) interajam com calendários, eventos e turmas de forma organizada.

🚀 Tecnologias e Bibliotecas Utilizadas

📚 Principais Bibliotecas

  • react → Biblioteca principal para construção da interface do sistema (SPA - Single Page Application)
  • react-dom → Responsável por renderizar os componentes React no navegador
  • react-router-dom → Gerenciamento de rotas e navegação entre páginas
  • @supabase/supabase-js → Integração com backend (banco de dados, autenticação e APIs)
  • sass → Pré-processador CSS para organização e reutilização de estilos
  • sweetalert2 → Exibição de alertas e modais estilizados (ex: sucesso, erro, confirmação)
  • date-fns → Manipulação e formatação de datas (utilizado no calendário)
  • react-datepicker → Componente de seleção de datas para formulários
  • lucide-react → Biblioteca de ícones utilizada na interface
  • xlsx → Leitura e exportação de arquivos Excel (ex: importação de alunos)

🧪 Bibliotecas de Teste

  • @testing-library/react → Testes de componentes React
  • @testing-library/jest-dom → Extensões para testes com DOM
  • @testing-library/user-event → Simulação de interações do usuário

⚙️ Ferramentas de Build

  • react-scripts → Scripts de build, execução e testes do projeto
  • dotenv → Gerenciamento de variáveis de ambiente
  • web-vitals → Monitoramento de performance da aplicação

Essas tecnologias foram escolhidas visando performance, escalabilidade e facilidade de manutenção do sistema.

🧱 Arquitetura do Projeto

O projeto segue uma estrutura modular baseada em React, separando responsabilidades por pastas:

  • components → Componentes reutilizáveis (menu, layout, calendário)
  • pages → Páginas principais do sistema
  • data → Dados estáticos (JSON)
  • utils → Funções auxiliares (ex: localStorage)
  • services → Integrações externas (ex: Supabase)
  • styles → Estilos globais

🔐 Controle de Acesso

O sistema possui controle de acesso baseado em tipo de usuário:

  • PrivateRoute → Protege rotas para usuários logados
  • RoleRoute → Restringe acesso por perfil

Tipos de usuários:

  • Aluno
  • Responsável
  • Professor
  • Coordenador

🧠 Lógica de Dashboard

function DashboardPage() {
  const usuario = JSON.parse(localStorage.getItem("usuario") || "null");
  const tipo = String(usuario?.tipo || "").toLowerCase();

  if (tipo === "aluno" || tipo === "responsavel") {
    return <DashboardAluno />;
  }

  return <Dashboard />;
}

O sistema identifica automaticamente o tipo de usuário logado e direciona para o dashboard correto.

💾 Armazenamento de Dados

Os dados podem vir de duas fontes:

  • JSON local → Dados fixos iniciais
  • LocalStorage → Eventos criados dinamicamente

Existe uma lógica de unificação desses dados para exibição no calendário.

📅 Calendário

O sistema possui diferentes visualizações de calendário:

  • Calendário lateral (resumo)
  • Calendário completo
  • Detalhes por dia

Eventos passados, atuais e futuros possuem diferenciação visual.
